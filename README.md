# NikVPN GAS Relay & VLESS (Codespace Edition)

این پروژه به شما امکان می‌دهد تا یک سرور رله (Relay) برای Google Apps Script و یک کانفیگ VLESS روی GitHub Codespaces راه‌اندازی کنید.

## ✨ ویژگی‌ها
- جایگزین رایگان برای Cloudflare Worker.
- رله کردن درخواست‌های GAS از طریق پروتکل Xray برای پنهان‌سازی بیشتر.
- ارائه لینک VLESS با ترانسپورت xHTTP.
- سیستم Keepalive با استفاده از tmux.

## 🚀 نحوه استفاده (گام‌به‌گام)

### مرحله ۱: فورک کردن و ساخت Codespace
۱. این ریپازیتوری را Fork کنید.
۲. روی دکمه **Code** کلیک کنید و تب **Codespaces** را انتخاب کنید.
۳. روی **Create codespace on main** کلیک کنید.
۴. منتظر بمانید تا کانتینر ساخته شود (ممکن است ۲-۳ دقیقه طول بکشد).

### مرحله ۲: راه‌اندازی سرویس‌ها
پس از باز شدن محیط Codespace، ترمینال را باز کرده و دستور زیر را اجرا کنید تا Xray و رله Node.js شروع به کار کنند:
```bash
bash .devcontainer/start.sh
```

### مرحله ۳: دریافت لینک و تنظیمات
بعد از اجرای دستور بالا، اطلاعات زیر در ترمینال نمایش داده می‌شود:
- **VLESS Link**: این لینک را در کلاینت خود (v2rayNG یا Nekobox) وارد کنید.
- **GAS Relay URL**: این آدرس را کپی کرده و در فایل `GAS-cod.Gs` جایگزین مقدار `WORKER_URL` کنید.

### مرحله ۴: تنظیم پورت‌ها (بسیار مهم)
۱. در پایین صفحه Codespace، تب **PORTS** را باز کنید.
۲. پورت‌های **443** و **8080** را پیدا کنید.
۳. روی هر کدام راست کلیک کرده و **Port Visibility > Public** را انتخاب کنید. (این کار برای دسترسی Google Apps Script الزامی است).

### مرحله ۵: زنده نگه داشتن Codespace
برای اینکه Codespace خاموش نشود، دستور زیر را در ترمینال اجرا کنید و اجازه دهید ترمینال باز بماند:
```bash
tmux attach -t nikvpn
```
برای خروج از نمای لاگ (بدون توقف سرویس)، کلیدهای **Ctrl+B** و سپس **D** را بزنید.

## ⚠️ نکات مهم
- هر ماه ۶۰ ساعت استفاده رایگان دارید. برای صرفه‌جویی، وقتی نیاز ندارید Codespace را Stop کنید.
- آدرس Codespace پس از هر بار ری‌استارت ممکن است تغییر کند، پس حتماً `show-link.sh` را برای دریافت آدرس جدید چک کنید.
